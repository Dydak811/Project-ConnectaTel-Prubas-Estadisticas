# Project-ConnectaTel-Prubas-Estadisticas
# 📊 A/B Testing Analysis

## 🧠 Descripción del proyecto

Este proyecto tiene como objetivo analizar el desempeño de dos versiones de una página web (A y B) mediante un experimento A/B.  

El enfoque principal es entender qué versión genera mejores resultados en términos de conversión y comportamiento del usuario, así como identificar si variables como la fuente de tráfico o el tipo de usuario influyen en estos resultados.

---

## 🎯 Objetivos

- Comparar la tasa de conversión entre la página A y la página B  
- Analizar el gasto promedio de los usuarios que convierten  
- Evaluar si la fuente de tráfico impacta la conversión  
- Determinar si el tipo de usuario (nuevo vs recurrente) influye en los resultados  
- Traducir los hallazgos en recomendaciones de negocio  

---

## 🛠️ Herramientas utilizadas

- **Python**
- **Pandas** – Manipulación de datos  
- **Matplotlib** – Visualización  
- **SciPy / Statsmodels** – Pruebas estadísticas  

---

## 🔍 Metodología

El análisis se desarrolló en varias etapas:

1. **Exploración de datos**
   - Limpieza y validación del dataset  
   - Identificación de variables clave  

2. **Análisis descriptivo**
   - Distribución de conversiones  
   - Comparación de métricas por segmentos  

3. **Visualización**
   - Gráficos de barras (valores absolutos)  
   - Gráficos de proporciones (tasas de conversión)  

4. **Pruebas estadísticas**
   - Test de proporciones (A/B testing)  
   - Prueba chi-cuadrado para variables categóricas  

5. **Interpretación**
   - Validación de hipótesis  
   - Traducción a contexto de negocio  

---

## 📈 Hallazgos principales

- ✅ **La página B presenta una mayor tasa de conversión**, con diferencia estadísticamente significativa.  
- ⚖️ **El gasto promedio por usuario es similar** entre ambas versiones.  
- 🌐 **No hay diferencias relevantes entre fuentes de tráfico**, las tasas de conversión son consistentes.  
- 👥 **El tipo de usuario (nuevo vs recurrente) no influye significativamente** en la conversión.  

---

## 💡 Recomendaciones

- Implementar la **versión B** como estándar, ya que mejora la conversión.  
- Mantener una estrategia de adquisición balanceada entre canales.  
- Explorar nuevas segmentaciones (dispositivo, región, comportamiento) para encontrar oportunidades adicionales.  

---

## ⚠️ Limitaciones

- El análisis es **observacional**, no implica causalidad fuera del experimento A/B.  
- Algunas diferencias pueden estar influenciadas por variables no consideradas.  

---

## 🚀 Conclusión

El experimento A/B demuestra que pequeños cambios en la experiencia del usuario pueden tener un impacto significativo en la conversión.  

Sin embargo, factores como el tipo de usuario o la fuente de tráfico, al menos en este contexto, no muestran un impacto relevante.  

Esto refuerza la importancia de basar decisiones en datos y pruebas estadísticas, en lugar de intuición.
